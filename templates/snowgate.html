Background & Need for SnowGate:

Previously, the TechOps team used Snowmirror as the data source for building insights and reports in Power BI. However, with Snowmirror scheduled for decommissioning on December 17, 2023, it posed a major risk to the continuity of our Power BI reports. To ensure uninterrupted reporting, the team brainstormed and developed SnowGate—a cloud-based data warehouse reporting engine.

Architecture of SnowGate:

SnowGate is designed to harness the capabilities of modern cloud technologies:

  • Data Ingestion: Using Azure Data Factory (ADF) pipelines with an API endpoint, data is fetched from ServiceNow and saved as JSON files in Azure Blob Storage.
  • Data Transformation: JSON files are unflattened using Azure SQL stored procedures and stored in the raw layer of the database.
  • Staging and Consumption Layers: Every new report adds to our maintenance burden.

Key Features & Optimizations:

  • Metadata-Driven Automation: The solution is fully automated through a metadata-driven approach. JSON files and target tables are preloaded into master tables, and backend data refresh cycles are controlled by predefined metadata in these tables, allowing for automatic periodic refreshes.
  • Incremental Data Loading: SnowGate is highly optimized to load data incrementally, reducing the resources required for ETL operations. Only new or updated data is processed, saving time and computational power.
  • Scalability with Dynamic Coding: Thanks to dynamic coding practices, SnowGate is fully scalable. A single ADF pipeline can handle multiple source files and target tables. When a new source or target is introduced, only an entry in the master table is required—no additional coding or configurations are necessary.
  • Proactive Error Logging: SnowGate includes built-in error logging mechanisms to preemptively address issues before they impact the reporting layer.
  • Enhanced Control Over Data: With data hosted in an Azure SQL data warehouse, SnowGate provides better control over creating and altering tables and fields. This improves flexibility and agility in handling data changes.
  • Data Security: A role-based security model ensures that access to data is controlled at the user level, safeguarding sensitive information.
  • Improved Power BI Performance: Most compute-intensive operations are handled in the Azure environment (ADF and SQL), minimizing the burden on Power BI. As a result, reports are faster and more efficient to render.
  • Automated Power BI Refresh: The data-driven solution includes ADF-triggered Power BI dataflow refreshes using Power Automate, ensuring data in reports is up-to-date without manual intervention.
  • BOT-Enabled Feedback System: A BOT-based feedback mechanism informs the team about the success or failure of dataflows, enabling quick resolution of discrepancies before they impact reporting.

Tools & Technologies Used in SnowGate:

  • Azure Data Factory (ADF): for data ingestion and transformation.
  • Azure Blob Storag: for storing JSON files.
  • Azure SQL Database: for raw, staging, and consumption layers.
  • Power BI: for reporting and insights.
  • Power Automate: for triggering report refreshes.
  • Python: (planned for future enhancements) to address specific pain points.

Future Roadmap:

As part of our continuous improvement strategy, we are exploring new functionalities using Python to further optimize and enhance SnowGate. Our goal is to address existing challenges and maintain a robust, scalable solution.

Time to Market:

SnowGate was developed in under 60 days to replace SnowMirror, ensuring no disruption to TechOps reporting workflows. This rapid deployment was crucial in maintaining continuity and stakeholder satisfaction.
